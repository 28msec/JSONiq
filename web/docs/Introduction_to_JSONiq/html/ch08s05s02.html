8.5.2. Treat expressions

Figure 8.30. TreatExpr

A treat expression checks that a JSONiq value matches a given sequence type. If it is not the case, an error is raised.
Example 8.84. Treat as expression
1 treat as integer
Result: 
1

Example 8.85. Treat as expression
1 treat as string
Result: 
An error was raised: "xs:integer" cannot be treated as type xs:string

Example 8.86. Treat as expression
"foo" treat as string
Result: 
"foo"

Example 8.87. Treat as expression
{ "foo" : "bar" } treat as object
Result: 
{ "foo" : "bar" }

Example 8.88. Treat as expression
({ "foo" : "bar" }, { "bar" : "foo" }) treat as json-item+
Result: 
{ "foo" : "bar" }
 { "bar" : "foo" }

Example 8.89. Treat as expression
[ 1, 2, 3 ] treat as array?
Result: 
[ 1, 2, 3 ]
