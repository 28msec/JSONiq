7.4. JSON representation of a Union Schema Type component

A Union Schema Type is represented by a JSON object. One of the pairs is named "$name" and specifies an optional name. The other pairs specify facets. If this object representation is $type, the represented Union Schema Type is as follows:
  • {name} : $type."$name" prefixed with Q{<namespace of the Schema which defines this type>}.
  • {facets} : $type entirely, but without the $name pair.
Examples


{
"$union" : [ "string", { "$array" : [ "integer" ] } ]
}
